# 审查报告输出模板

## 报告头部

```markdown
# 需求完成度审查报告

**需求文档**：{文档名称/路径}
**项目路径**：{项目根目录}
**审查日期**：{YYYY-MM-DD}
**审查人**：Codex / requirement-audit

---
```

## 总体摘要

```markdown
## 总体摘要

| 需求点总数 | 已完成 | 疑似已完成 | 部分完成 | 未完成 | 无法验证 |
|-----------|--------|-----------|---------|--------|---------|
| {N}       | {n}    | {n}       | {n}     | {n}    | {n}     |

**完成率**：{metric}（已完成 + 疑似已完成）/ 总数

**高风险项**：{n} 项（存在潜在 Bug 的已完成项）

**结论**：{一句话总体判断}
```

## 逐项审查

```markdown
## 逐项审查

### [R{id}] {需求点标题}

**需求描述**：{原文或概括}

**完成度**：已完成 / 疑似已完成 / 部分完成 / 未完成 / 无法验证

**证据链**：

- {证据项1：文件路径 + 行号 + 说明}
- {证据项2：配置表名 + 列名/行ID + 说明}
- {证据项3：调用链说明}

**缺失/不足**（如非已完成）：

- {缺失项1}
- {缺失项2}

**潜在 Bug 风险**（如适用）：

| 风险等级 | 风险描述 | 触发条件 | 游戏内表现 |
|---------|---------|---------|-----------|
| 高/中/低 | {描述}   | {条件}   | {表现}     |

---
```

## Bug 风险汇总

```markdown
## Bug 风险汇总

| 优先级 | 关联需求 | 风险描述 | 影响范围 |
|-------|---------|---------|---------|
| P0    | R{n}   | {描述}   | {范围}   |
| P1    | R{n}   | {描述}   | {范围}   |
```

## 搜索覆盖统计

```markdown
## 搜索覆盖统计

| 需求点 | 中文搜索 | 英文搜索 | 配表搜索 | 资源搜索 | 命中文件数 |
|-------|---------|---------|---------|---------|-----------|
| R{n}  | 已执行/未执行    | 已执行/未执行    | 已执行/未执行    | 已执行/未执行    | {n}       |

**总搜索关键词数**：{n}
**总命中文件数**：{n}
**读取的配置表数**：{n}
```

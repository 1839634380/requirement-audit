# Bug 风险检查规则 (Bug Risk Analysis Rules)

## 通用风险检查清单

### 1. 边界条件

| 检查项 | 检查方法 | 风险信号 |
|-------|---------|---------|
| 空值/Null 处理 | 搜索函数中对指针/引用的判空 | 缺少 `if (ptr)` / `nullptr` 检查 |
| 数组越界 | 检查数组/容器访问前是否有 size 检查 | 直接 `arr[i]` 而无 `i < arr.Num()` |
| 极端数值 | 检查整数溢出、浮点精度 | 无范围的累加/乘法运算 |
| 默认值缺失 | 检查配置读取时是否有 fallback | `GetValue()` 无默认参数 |

### 2. 状态一致性

| 检查项 | 检查方法 | 风险信号 |
|-------|---------|---------|
| 初始化 | 构造函数/BeginPlay 中字段初始化 | 成员变量未初始化 |
| 重置/清理 | EndPlay/Reset/OnDestroy | 状态残留、未解绑事件 |
| 状态机完整性 | 枚举值与 switch/if-else 覆盖 | 未处理的枚举值分支 |
| 数据同步 | 客户端/服务器字段同步 | 缺少 Replicated/OnRep |

### 3. 并发/时序

| 检查项 | 检查方法 | 风险信号 |
|-------|---------|---------|
| 多次触发 | 检查事件注册是否有去重 | 重复 Bind/Dynamic 委托 |
| 事件顺序 | 检查初始化顺序依赖 | 跨模块 Init 顺序假定 |
| 定时器残留 | 搜索 SetTimer/ClearTimer | 有 Set 无对应 Clear |
| 竞态条件 | 检查多线程/异步回调中的共享状态 | 无锁/无原子操作的写操作 |

### 4. 配置完整性

| 检查项 | 检查方法 | 风险信号 |
|-------|---------|---------|
| 配置表存在性 | 搜索配表名 | 代码引用的表不存在 |
| 行完整性 | 读取配置表内容 | 代码引用的 key 在表中不存在 |
| 列完整性 | 检查列的导出配置 | 列未导出到对应端（c/s/p） |
| 引用完整性 | 检查跨表引用 link | 引用链断裂 |

### 5. 数据持久化

| 检查项 | 检查方法 | 风险信号 |
|-------|---------|---------|
| 存档字段 | 搜索 SaveGame/Serialize | 新增字段未标记序列化 |
| 版本兼容 | 检查存档版本号处理 | 无版本号或迁移逻辑 |
| 数据恢复 | 检查读档后的状态恢复 | 读档后未重新计算依赖状态 |

### 6. UI/表现层

| 检查项 | 检查方法 | 风险信号 |
|-------|---------|---------|
| UI 刷新 | 检查数据变化后的 UI 更新 | 数据变化后未调用 UI 刷新 |
| 输入阻断 | 检查弹窗/动画期间的输入处理 | 弹窗时未禁用底层交互 |
| 资源加载 | 检查同步/异步加载模式 | 同步加载大资源导致卡顿 |
| 多分辨率 | 检查 UI 布局方式 | 绝对坐标而非锚点/相对布局 |

## 风险等级定义

| 等级 | 定义 | 示例 |
|------|------|------|
| **P0 (高)** | 必现崩溃/数据丢失/功能完全不可用 | 空指针未判空导致崩溃 |
| **P1 (中)** | 特定条件下功能异常/数据错误 | 边界情况下状态机卡死 |
| **P2 (低)** | UI 表现不完美/非核心路径问题 | 极端分辨率下文字重叠 |

## 游戏内表现推演模板

对每个发现的风险，按以下模板推演：

```
触发条件：{玩家操作 / 系统触发 / 时间触发}
现象：{玩家看到什么 / 发生什么}
影响：{功能级别影响 / 体验级别影响}
复现步骤：
1. {步骤1}
2. {步骤2}
3. {期望 vs 实际}
```
